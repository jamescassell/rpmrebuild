RPMREBUILD

 Commentaires..

resume

rpmrebuild est un logiciel qui permet de construire des packages rpm depuis des packages rpm installés

utilisation

  • depuis la version 0.5, il marche sur toute distribution utilisant rpm
  • il ne necessite pas de connaissances sur la fabrication de packages

reconstruire un rpm perdu

C'etait le premier objectif : automatiser la fabrication d'un fichier rpm depuis la base de donnée rpm, quand on a perdu le fichier d'origine.

modifier un package

Vous avez modifié un logiciel (fichier de configuration, scripts ...) et vous voulez le distribuer (au format rpm) tel quel : c'est maintenant mon utilisation principale.

conseils d'utilisation

  • rpmrebuild ne necessite pas d'etre utilise sous l'identité "root", mais cela peut etre necessaire pour acceder aux fichiers protégés
  • le fichier "spec" est détruit après utilisation. Pour le conserver, utiliser l'option -s
  • les fichiers rpm sont construits dans le repertoire naturel (pour rpm) : par exemple, sur RedHat, par defaut sur /usr/src/redhat/RPMS
    mais vous pouvez le modifier avec l'option -d ou avec un fichier ~/.rpmmacros (ligne "%_topdir un_repertoire")
  • rpmrebuild a de nombreuses autres options, consultez-les avec man rpmrebuild ou rpmrebuild --help

autres projets
